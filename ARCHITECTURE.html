Node.js + React + MySQL 기반 사내 자산 통합 관리 플랫폼

3. 사용자 역할 체계

관리자 (admin) — 전체 시스템 제어 권한. 모든 부서의 자산 등록/수정/폐기, 사용자 관리, 대여 승인/거절, 대시보드 및 전체 통계 조회가 가능합니다.
부서장 (manager) — 자기 부서 내 자산 등록/수정, 부서원의 대여 요청 승인/거절, 부서 자산 대시보드 및 통계 조회가 가능합니다.
일반 사용자 (user) — 자기 부서 자산 조회, 자산 대여 요청 및 반납, 본인 알림 확인이 가능합니다.

4. 데이터베이스 ERD

테이블설명주요 컬럼
departments부서 정보name, code
users사용자 계정email, role (admin/manager/user), department_id
asset_categories자산 분류 (계층 구조)name, parent_id
assets자산 마스터asset_code, name, status, purchase_cost, assigned_to
asset_assignments대여/반납 기록asset_id, user_id, status, checkout_date, actual_return
approval_workflows승인 워크플로우action_type, requester_id, approver_id, status
notifications알림user_id, title, message, is_read
asset_logs자산 변경 이력asset_id, action, details (JSON)

5. API 엔드포인트

메서드경로설명권한
POST/api/auth/login로그인공개
GET/api/auth/me내 정보 조회로그인
GET/api/assets자산 목록 (필터/검색/페이지네이션)로그인
GET/api/assets/:id자산 상세로그인
POST/api/assets자산 등록관리자/부서장
PUT/api/assets/:id자산 수정관리자/부서장
DEL/api/assets/:id자산 폐기 처리관리자
POST/api/assignments/request대여 요청로그인
PUT/api/assignments/:id/approve대여 승인/거절관리자/부서장
PUT/api/assignments/:id/return반납 처리로그인
GET/api/dashboard/stats대시보드 통계관리자/부서장
GET/api/notifications내 알림 목록로그인
GET/api/users사용자 목록관리자

7. 빠른 시작 가이드

Docker로 실행 (권장)

# 1. 프로젝트 폴더에서 실행
docker-compose up -d

# 2. DB 스키마 자동 적용 후, 브라우저에서 접속
#    프론트엔드: http://localhost:3000
#    API 서버:   http://localhost:4000

# 3. 기본 관리자 계정
#    이메일: admin@company.com
#    비밀번호: admin123 (초기 비밀번호, 변경 필요)

로컬 개발 환경

# 백엔드
cd backend
cp .env.example .env   # DB 정보 수정
npm install
npm run dev

# 프론트엔드 (별도 터미널)
cd frontend
npm install
npm start

9. 향후 확장 포인트

QR코드/바코드 스캔 — 모바일에서 자산 태그를 스캔하여 빠른 조회/대여 처리

엑셀 일괄 등록/내보내기 — 대량 자산을 CSV/Excel로 가져오기 및 리포트 다운로드

이메일 알림 연동 — Nodemailer를 통한 반납 기한 알림, 승인 요청 이메일 발송

감가상각 자동 계산 — 정액법/정률법 기반 자산 가치 감소 자동 산출

SSO 연동 — 사내 LDAP/Active Directory 또는 Google/Microsoft OAuth 연동
